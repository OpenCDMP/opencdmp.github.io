# Introduction

There are a lot of views in the application available only for users that are given elevated privileges. In the pages that follow we will discover all the configuration options available in these views for an administrator, with the most important being the following:
- **[Plan Blueprints](/docs/administration/blueprints/index.md)**: Being able to control the blueprints available for [plans](/docs/application/plans/index.md).
- **[Description Templates](/docs/administration/templates/index.md)**: Being able to control the templates available for [descriptions](/docs/application/descriptions/index.md).
- **[Description Template Types](/docs/administration/template-types.md)**: Being able to control the description types available for [description templates](/docs/application/templates.md).
- **[Prefilling Sources](/docs/administration/prefilling-sources.md)**: Being able to configure external data sources that are made available to filling out a description.
- **[Reference Types](/docs/administration/reference-types.md)**: Being able to configure static or external data sources that are made available as autocomplete fields on the forms of the platform.
- **[Plan Statuses](/docs/administration/plan-statuses.md)**: Being able to control plan statuses.
- **[Description Statuses](/docs/administration/description-statuses.md)**: Being able to control description statuses.
- **[Users](/docs/administration/users.md)**: Being able to control the users of the platform.
- **[Tenants](/docs/administration/tenants.md)**: Being able to control the tenants of the platform. Think of tenants as different organisations that can use the application independently at the same time without the need of different deployments.
- **[Tenant Users](/docs/administration/tenant-users.md)**: Being able to control the users that exist in tenant thath your are logged in with.
- **[Notification Templates](/docs/administration/notification-templates.md)**: Being able to modify the structure of notifications sent to users based on different events.
- **[Anotation statuses](/docs/administration/annotation-statuses.md)**: Being able to control annotation statuses.
- **[Languages](/docs/administration/languages.md)**: Being able to add support for more languages or modify existing ones.
- **[Supportive Material](/docs/administration/supportive-material.md)**: Being able to add support or modify supportive materials.
- References: Being able to view and edit [references](/docs/application/references.md) that have been used in application.
- **[Usage limits](/docs/administration/usage-limits.md)**: Being able to control usage limits.
- **[Maintenance](/docs/administration/maintenance.md)**: Being able to maintenance application.
